# 요구사항 명세서 — NingNing AI 운세 서비스

---

## 1. 문서 개요

### 1.1 목적
본 문서는 NingNing AI 운세 서비스의 MVP 버전에 대한 기능적/비기능적 요구사항을 정의한다. 개발팀이 구현해야 할 범위와 기준을 명확히 하여, 기획 의도와 개발 결과물 간의 간극을 최소화한다.

### 1.2 범위
- MVP 1차 출시 범위에 한정
- 후순위 기능(궁합, 타로, 전문가 상담 등)은 본 문서에서 제외

### 1.3 용어 정의
| 용어 | 정의 |
|------|------|
| 사주 | 생년월일시를 기반으로 한 동양 명리학 체계 |
| 만세력 | 음양력 변환 및 천간지지 계산 체계 |
| 오행 | 목(木)·화(火)·토(土)·금(金)·수(水) 다섯 가지 기운 |
| 일간 | 사주 팔자 중 본인을 나타내는 천간 |
| 3축 해석 | 연애·일·재물 세 가지 주제별 해석 |
| 심층 리포트 | 유료 상세 해석 콘텐츠 |
| AI 추가질문 | 사용자가 현재 상황을 붙여 추가로 묻는 기능 |

## 2. 기능 요구사항

### 2.1 사주 입력 (FR-001)

#### FR-001-1: 생년월일 입력
- **설명**: 사용자가 생년월일을 입력할 수 있다
- **입력 항목**:
  - 양력/음력 선택 (필수, 기본값: 양력)
  - 윤달 여부 (음력 선택 시 표시)
  - 생년 (필수, 1940~2010년)
  - 생월 (필수, 1~12월)
  - 생일 (필수, 해당 월 유효 범위)
- **우선순위**: 필수
- **검증**: 유효하지 않은 날짜 입력 시 에러 메시지 표시

#### FR-001-2: 출생시간 입력
- **설명**: 사용자가 출생시간을 입력할 수 있다
- **입력 항목**:
  - 출생시간 선택 (선택, 시간대 목록 또는 "모름" 선택 가능)
  - 시간대: 자시(23:30~01:30), 축시(01:30~03:30), ... , 해시(21:30~23:30)
- **예외 처리**: "모름" 선택 시에도 납득 가능한 결과를 생성해야 함
- **우선순위**: 필수

#### FR-001-3: 성별 입력
- **설명**: 사용자가 성별을 선택할 수 있다
- **입력 항목**: 남성/여성 (필수)
- **우선순위**: 필수

#### FR-001-4: 입력 유효성 검증
- **설명**: 모든 필수 항목이 입력되지 않으면 다음 단계로 진행 불가
- **검증 규칙**:
  - 양/음력 미선택 시 안내 메시지
  - 생년월일 범위 초과 시 에러
  - 음력 선택 시 양력 자동 변환 정확도 보장
- **우선순위**: 필수

---

### 2.2 사주 계산 엔진 (FR-002)

#### FR-002-1: 만세력 계산
- **설명**: 입력된 생년월일시를 기반으로 사주 팔자를 계산한다
- **처리 항목**:
  - 양력↔음력 변환
  - 천간(天干) 10개, 지지(地支) 12개 매핑
  - 년주, 월주, 일주, 시주 산출
- **정확도**: 1940~2010년 범위 내 100% 정확도 목표
- **우선순위**: 필수

#### FR-002-2: 오행 분석
- **설명**: 사주 팔자에서 오행 분포를 분석한다
- **산출 항목**:
  - 오행별 개수 및 비율
  - 일간(日干) 기준 강약 판단
  - 용신(用神) 추정
- **우선순위**: 필수

#### FR-002-3: 오늘의 운세 계산
- **설명**: 사주 원국과 오늘의 일진(日辰)을 조합하여 오늘의 흐름을 산출한다
- **산출 항목**:
  - 오늘 일진과 사주 원국의 관계
  - 3축(연애/일/재물) 흐름 판단
  - 주의/행운 포인트
- **우선순위**: 필수

---

### 2.3 AI 해설 생성 (FR-003)

#### FR-003-1: 오늘의 총평 생성
- **설명**: 사주 계산 결과를 바탕으로 오늘의 한 줄 총평을 생성한다
- **출력 형식**:
  - 한 줄 총평 (30~60자)
  - 오늘의 포인트 키워드 1~2개
- **톤**: 담백하지만 한 줄은 강하게, 확정형이 아닌 가능성 언어
- **우선순위**: 필수

#### FR-003-2: 3축 해석 카드 생성
- **설명**: 연애·일·재물 각 축에 대한 해석 카드를 생성한다
- **출력 형식** (축당):
  - 주제 아이콘/라벨
  - 해석 문장 (80~150자)
  - 행동 조언 1개 (30~60자)
- **우선순위**: 필수

#### FR-003-3: AI 해설 전체 구조
- **설명**: 결과 화면에 표시되는 AI 해설의 전체 구조
- **권장 출력 순서**:
  1. 오늘의 한 줄 총평
  2. 왜 이런 해석이 나왔는지 배경 설명
  3. 연애·일·재물 3축 해석
  4. 오늘 하라 / 피하라 행동 조언
  5. 행운 단서 1~2개
  6. "지금 상황을 붙여 더 물어보기" CTA
- **우선순위**: 필수

#### FR-003-4: 가드레일
- **설명**: AI 해설 생성 시 반드시 준수해야 할 제한 사항
- **금지 항목**:
  - 의료·법률·자해 관련 조언
  - 특정 종목/금액을 지정한 투자·매매 지시 (사주 흐름 기반 재물운·승부운 해석은 허용)
  - 공포 조장, 확정적 미래 단정
  - 불안을 이용한 판매 유도 문구
  - 특정 종교/사상 비하
- **허용 항목**:
  - 사주 오행 기반 행운 숫자, 행운 색상, 행운 방위 제시
  - 도박/게임 승부운 질문에 대한 사주 맥락 답변 (면책 문구 포함)
  - 연애/고백/이별 등 관계 관련 사주 해석
- **필수 준수**:
  - 결과는 단정형이 아닌 가능성 언어 사용
  - "참고용 해석"임을 명시할 수 있는 구조
  - 동일 입력에 대해 문체 일관성 유지
  - 메시지 길이 예측 가능
  - 3축(연애/일/재물) 해석이 서로 유사하지 않도록 차별화
- **우선순위**: 필수

---

### 2.4 AI 추가질문 (FR-004)

#### FR-004-1: 추가질문 입력
- **설명**: 사용자가 현재 상황을 텍스트로 입력하여 추가 질문할 수 있다
- **입력 제한**:
  - 텍스트 최대 500자
  - 무료: 2회 제한
  - 유료(질문 팩): 추가 횟수 확장 (결제 연동 후)
- **우선순위**: 필수

#### FR-004-2: 추가질문 답변 생성
- **설명**: 사용자의 추가 질문에 대해 사주 맥락을 유지하며 답변한다
- **답변 구조**:
  - 현재 상황 이해 표현
  - 사주 맥락 기반 해석
  - 행동 조언
- **맥락 유지**: 이전 사주 계산 결과와 총평을 참조하여 일관된 답변
- **우선순위**: 필수

#### FR-004-3: 질문 횟수 관리
- **설명**: 무료/유료 질문 횟수를 관리한다
- **동작**:
  - 무료 횟수 소진 시 "질문 팩" 결제 유도 메시지
  - 남은 질문 횟수 표시
  - 세션 기반 또는 일 기반 초기화 (실험을 통해 결정)
- **우선순위**: 필수

---

### 2.5 심층 리포트 (FR-005)

#### FR-005-1: 심층 리포트 생성
- **설명**: 결제 완료 후 상세 심층 리포트를 생성한다
- **리포트 내용**:
  - 사주 원국 구조 요약 (오행 분포, 일간 특성)
  - 오늘의 흐름 상세 해석
  - 3축별 심층 해석 (각 200~400자)
  - 배경 설명 (왜 이런 흐름인지)
  - 구체적 행동 조언 3~5개
  - 이번 주/이번 달 흐름 미리보기
- **우선순위**: 필수

#### FR-005-2: 리포트 저장
- **설명**: 생성된 리포트를 저장하여 재열람할 수 있다
- **동작**:
  - 결제 완료된 리포트는 자동 저장
  - 마이페이지에서 과거 리포트 목록 조회
  - 리포트 재열람 기능
- **보관 기간**: 최소 90일 (추후 확장)
- **우선순위**: 필수

---

### 2.6 결제 시스템 (FR-006)

#### FR-006-1: 결제 처리
- **설명**: 심층 리포트 및 질문 팩에 대한 결제를 처리한다
- **결제 수단**: PG사 연동 (카드, 간편결제)
- **상품 유형**:
  - 심층 리포트: 3,900 ~ 7,900원 (단건)
  - AI 질문 팩: 1,900 ~ 4,900원 (단건)
- **우선순위**: 필수

#### FR-006-2: 결제 실패 처리
- **설명**: 결제 실패 시 사용자에게 안내하고 재시도를 유도한다
- **동작**:
  - 결제 실패 원인 안내 (카드 한도, 네트워크 오류 등)
  - 재시도 버튼
  - 다른 결제 수단 안내
- **우선순위**: 필수

#### FR-006-3: 환불 처리
- **설명**: 환불 요청에 대한 처리 동선
- **정책**: 리포트 미열람 시 전액 환불, 열람 후 환불 불가 (상세 정책 별도 수립)
- **동작**: 문의 채널 안내 (이메일/폼)
- **우선순위**: 필수

---

### 2.7 랜딩/온보딩 (FR-007)

#### FR-007-1: 메인 랜딩 페이지
- **설명**: 서비스 소개 및 사주 입력 유도. 비회원/회원에 따라 화면이 분기됨.
- **구성**:
  - GNB 헤더: 좌측 "NingNing" 로고, 우측 로그인/마이페이지
  - 한 줄 소개: "사주명리학 기반 AI 오늘의 운세"
  - 서브카피: "정통 사주 계산과 AI 해석으로 오늘 하루를 위한 현실적인 가이드를 제공합니다"
  - 특징 소개 3가지 (정통 사주 계산, AI 맞춤 해석, 안전한 운세)
  - CTA 버튼: 비회원 "무료로 운세 보기" / 회원 "오늘의 운세 보기"
  - 테마별 랜딩 링크 (회원만 표시)
- **우선순위**: 필수

#### FR-007-2: 로그인 페이지
- **설명**: 이메일/비밀번호 로그인 및 소셜 로그인 제공. 회원가입은 별도 페이지로 분리.
- **구성**:
  - 설명 문구: "내 운세일기를 만들어보세요"
  - 이메일/비밀번호 로그인 폼
  - 회원가입 유도 링크 → `/signup`
  - 소셜 로그인 (카카오, Google)
- **우선순위**: 필수

#### FR-007-3: 회원가입 페이지
- **설명**: 이메일 OTP 인증 코드 기반 회원가입 처리
- **구성**:
  - 이메일 입력 + OTP 6자리 코드 발송/검증 (Supabase signUp → verifyOtp)
  - 닉네임 입력 (2~20자, user_metadata.name에 저장)
  - 비밀번호 + 비밀번호 확인 (8자 이상)
  - 이용약관/개인정보 처리방침 동의 (모달로 내용 표시)
  - 가입 성공 시 자동 로그인 후 홈으로 이동
- **우선순위**: 필수

#### FR-007-4: 테마별 랜딩 페이지
- **설명**: 검색 유입용 테마별 랜딩 (2~3종)
- **테마**: 연애운, 이직운, 재물운
- **구성**: 테마 맞춤 카피 + 샘플 문장 + 입력 유도
- **우선순위**: 초기 개선

### 2.8 사용자 관리 (FR-008)

#### FR-008-1: 저장/재열람
- **설명**: 결과 및 리포트를 저장하고 다시 볼 수 있다
- **방식**: 브라우저 로컬 저장 또는 간편 로그인 후 서버 저장
- **우선순위**: 필수

#### FR-008-2: 개인정보 관리
- **설명**: 사용자가 자신의 데이터를 관리할 수 있다
- **기능**:
  - 저장된 사주 정보 확인
  - 데이터 삭제 요청
  - 개인정보 처리방침 열람
- **우선순위**: 필수

#### FR-009-1: 결과 공유 이미지

- **공유 채널**: 카카오톡, 인스타그램 스토리, 링크 복사
- **우선순위**: 초기 개선

#### FR-010-1: 프롬프트/카피 관리
- **설명**: 관리자가 AI 프롬프트 및 서비스 카피를 수정할 수 있다
- **기능**: 프롬프트 버전 관리, 카피 수정, 변경 이력 조회
- **우선순위**: 초기 개선

- **우선순위**: 초기 개선

## 3. 비기능 요구사항

### 3.1 성능 (NFR-001)
| 항목 | 요구사항 |
|------|----------|
| 결과 생성 시간 | 5초 이내 (사주 계산 + AI 해설 생성) |
| 페이지 로딩 | 초기 로딩 3초 이내 (3G 환경 제외) |
| 동시 사용자 | 최소 100명 동시 접속 지원 |
| AI 응답 시간 | 추가질문 답변 3초 이내 |

### 3.2 가용성 (NFR-002)
| 항목 | 요구사항 |
|------|----------|
| 서비스 가용률 | 99% 이상 (월 기준) |
| 장애 복구 | 주요 장애 발생 시 1시간 이내 복구 |
| 백업 | 사용자 데이터 일 1회 백업 |

### 3.3 보안 (NFR-003)

|------|----------|

## 6. 우선순위 분류

### 필수 (Must Have) — MVP 출시 포함
- ✅ 사주 입력 (양/음력, 생년월일, 출생시간, 성별)
- ✅ 사주 계산 엔진 (만세력, 오행, 일진)
- ✅ 오늘의 총평 + 3축 해석 카드
- ✅ AI 추가질문 2회
- ✅ 심층 리포트 생성/조회 (결제 게이트 미연동)
- ✅ 저장/재열람 (localStorage + 마이페이지)
- ✅ 기본 이벤트 로그 (코드 준비, GA4 연동 대기)
- ✅ 개인정보 처리방침
- ⏳ 결제/환불 동선 (사업자 등록 후)

### 초기 개선 (Should Have) — 출시 후 2~4주
- ✅ 테마별 랜딩 2종 (연애운, 이직운)
- ✅ 공유 이미지 카드 (Canvas API 기반 이미지 생성 + 링크 복사)
- ⏳ 알림 옵트인
- ⏳ 관리자용 카피 수정 도구
- ⏳ 후기 수집

### 후순위 (Won't Have for MVP)
- 궁합 서비스
- 신년운세 패키지
- 타로/관상
- 전문가 상담 연결
- 앱 패키징
- 커뮤니티/댓글
- 무제한 채팅
- 포인트 적립/출석체크
