# NingNing 사이트맵

> 최종 갱신: 2026-04-05

## 전체 구조

```
/                          ← 메인 랜딩 페이지
├── /landing/[theme]       ← 테마별 랜딩 (love, career)
├── /login                 ← 로그인
├── /signup                ← 회원가입
├── /input                 ← 생년월일시 입력
├── /result                ← 운세 결과
│   ├── /question          ← AI 추가질문 (채팅)
│   └── /report            ← 심층 리포트
├── /my                    ← 마이페이지 (로그인 필요)
├── /terms                 ← 이용약관
└── /privacy               ← 개인정보처리방침
```

## 사용자 흐름 (User Flow)

```
[신규 사용자]
  메인(/) → 입력(/input) → 결과(/result) → 추가질문(/question) 또는 리포트(/report)

[테마 유입]
  랜딩(/landing/love) → 입력(/input) → 결과(/result)

[회원가입]
  메인(/) → 로그인(/login) → 회원가입(/signup) → 자동 로그인 → 메인(/)

[회원 사용자]
  메인(/) → 로그인(/login) → 메인(/) → 입력(/input) → 결과(/result) → 마이페이지(/my)
```

## 공통 레이아웃

- **RootLayout** (`layout.tsx`): Geist 폰트, AuthProvider 래핑, `lang="ko"`
- **배경**: `bg-zinc-50 dark:bg-black`
- **콘텐츠 폭**: `max-w-md` (모바일 퍼스트)
- **다크모드**: 시스템 설정 자동 반영

## 데이터 흐름

```
[sessionStorage]
  sajuInput  ← /input에서 저장 → /result, /question, /report에서 읽기
  sajuResult ← /result에서 저장 → /question, /report에서 읽기

[localStorage]
  savedResults ← /result에서 저장 → /my에서 읽기/삭제

[API 호출]
  /api/saju      ← /result에서 호출 (사주 계산)
  /api/interpret  ← /result에서 호출 (AI 해석)
  /api/question   ← /question에서 호출 (추가질문 답변)
  /api/report     ← /report에서 호출 (심층 리포트)
```
