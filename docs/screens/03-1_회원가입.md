# 03-1. 회원가입 페이지

> 경로: `/signup` | 파일: `src/app/signup/page.tsx` | 렌더링: Client Component

## 역할

이메일 인증 코드 기반 회원가입을 처리합니다. `/login`에서 "회원가입" 링크를 통해 진입합니다.

## 화면 구성

### 1) 헤더
- "← 뒤로가기" → `router.back()`

### 2) 이메일 입력 + 코드 발송
- **이메일 입력**: type=email, placeholder="email@example.com"
- **확인 버튼**: 이메일 입력 우측에 배치
  - 클릭 시 해당 이메일로 인증 코드 발송
  - 발송 중: "발송 중..." (비활성화)
  - 발송 완료: "재발송" 으로 텍스트 전환

### 3) 인증 코드 입력 + 검증
- **코드 입력**: 이메일 확인 후 노출
- **확인 버튼**: 코드 입력 우측에 배치
- **결과 메시지**:
  - 성공: 초록색 "인증되었습니다"
  - 실패: 빨간색 에러 이유 표시 (예: "인증 코드가 올바르지 않습니다", "인증 코드가 만료되었습니다")

### 4) 비밀번호 입력
- **비밀번호**: type=password, placeholder="8자 이상 입력하세요", minLength=8
- **비밀번호 확인**: type=password, placeholder="비밀번호를 다시 입력하세요"
- **불일치 메시지**: 빨간색 "비밀번호가 일치하지 않습니다" (입력 중 실시간 비교)

### 5) 이용약관 동의
- **이용약관 동의** (필수): 체크박스 + "이용약관에 동의합니다" + 보기 링크 → 모달로 약관 내용 표시
- **개인정보 처리방침 동의** (필수): 체크박스 + "개인정보 처리방침에 동의합니다" + 보기 링크 → 모달로 방침 내용 표시
- 모달: 반투명 오버레이 + 스크롤 가능한 본문 + "확인" 닫기 버튼

### 6) 회원가입 버튼
- **텍스트**: "회원가입" / "처리 중..."
- **스타일**: h-12, w-full, rounded-xl, bg-foreground
- **비활성화 조건**: 이메일 미인증 / 비밀번호 미입력 / 비밀번호 불일치 / 약관 미동의

## 상태 관리
- `email`: 이메일 입력값
- `verificationCode`: 인증 코드 입력값
- `emailSent`: 코드 발송 완료 여부
- `emailVerified`: 이메일 인증 완료 여부
- `verificationError`: 인증 에러 메시지
- `password`: 비밀번호
- `passwordConfirm`: 비밀번호 확인
- `agreeTerms`: 이용약관 동의 여부
- `agreePrivacy`: 개인정보 처리방침 동의 여부
- `loading`: 제출 중 상태

## 이동 경로

| 요소 | 이동 대상 |
|------|-----------|
| ← 뒤로가기 | 이전 페이지 (router.back) |
| 이용약관 보기 | 모달 (페이지 이동 없음) |
| 개인정보 처리방침 보기 | 모달 (페이지 이동 없음) |
| 가입 성공 | 자동 로그인 후 `/` |
