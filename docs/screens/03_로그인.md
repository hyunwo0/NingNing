# 03. 로그인 / 회원가입 페이지

> 경로: `/login` | 파일: `src/app/login/page.tsx` | 렌더링: Client Component

## 화면 구성

### 1) 뒤로가기 링크
- "← 홈으로" → `/`

### 2) 타이틀 영역
- **로고**: 12x12 검은 사각형 안에 "N" 글자
- **제목**: 모드에 따라 "로그인" 또는 "회원가입"
- **설명**:
  - 로그인 모드: "운세 결과를 저장하고 관리하세요"
  - 회원가입 모드: "이메일과 비밀번호로 가입하세요"

### 3) 로그인/회원가입 탭
- 2개 탭 (로그인 | 회원가입)
- 스타일: rounded-xl muted 배경, 선택된 탭은 흰 배경 + 그림자

### 4) 이메일/비밀번호 폼
- **이메일 입력**: type=email, placeholder="email@example.com"
- **비밀번호 입력**: type=password
  - 회원가입 시 placeholder="8자 이상 입력하세요", minLength=8
  - 로그인 시 placeholder="비밀번호 입력"
- **에러 메시지**: 빨간색 텍스트
- **제출 버튼**: h-12, rounded-xl, bg-foreground
  - 텍스트: "로그인" / "가입하기" / "처리 중..."

### 5) 구분선
- "또는" 텍스트 + 양쪽 가로선

### 6) 소셜 로그인 버튼
- **카카오**: 노란 배경(#FEE500), 검은 텍스트, 카카오 로고 SVG
  - "카카오로 시작하기"
  - scope: `profile_nickname profile_image`
- **Google**: 흰 배경 + 테두리, Google 로고 SVG (4색)
  - "Google로 시작하기"
- 소셜 로그인 리다이렉트: `/auth/callback`

### 7) 하단 안내
- "로그인 없이도 운세를 볼 수 있습니다"

## 회원가입 성공 화면
- 초록 체크 아이콘 (16x16 원형)
- "가입 완료!" 타이틀
- "입력하신 이메일로 확인 메일을 보냈습니다. / 메일의 링크를 클릭하면 가입이 완료됩니다."
- "로그인하기" 버튼 → 로그인 모드로 전환

## 상태 관리
- `mode`: 'login' | 'signup'
- `email`, `password`: 폼 입력값
- `error`: 에러 메시지
- `loading`: 제출 중 상태
- `signupSuccess`: 가입 성공 화면 표시

## 인증 처리
- Supabase Auth 사용 (`getSupabaseBrowserClient()`)
- 로그인 성공 → `router.push('/')` + `router.refresh()`
- 에러 한글화: "Invalid login credentials" → "이메일 또는 비밀번호가 올바르지 않습니다"

## 이동 경로

| 요소 | 이동 대상 |

| ← 홈으로 | `/` |
| 로그인 성공 | `/` |
| 소셜 로그인 | `/auth/callback` → `/` |
