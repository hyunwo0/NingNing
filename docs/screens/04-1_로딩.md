# 04-1. 로딩 페이지

> 경로: `/loading?type=saju|report` | 파일: `src/app/loading-screen/page.tsx` | 렌더링: Client Component

## 역할

API 호출 중 재미있는 로딩 문구를 보여주는 공통 페이지. 쿼리 파라미터 `type`에 따라 문구와 API/이동 대상이 달라집니다.

## 레이아웃

```
┌─────────────────────────────────┐
│                                 │
│                                 │
│                                 │
│              ◌                  │ ← 스피너
│                                 │
│   사주 팔자를 펼치는 중...       │ ← 문구 (2초 간격 순환)
│                                 │
│                                 │
│                                 │
└─────────────────────────────────┘
```

## 타입별 설정

### type=saju (사주 계산 + AI 해석)
- **진입**: `/input`에서 CTA 클릭
- **API**: `POST /api/saju` → `POST /api/interpret` (순차 호출, AI 해석 실패 시 무시)
- **완료 후 이동**: `/result`
- **문구**:
  - "사주 팔자를 펼치는 중..."
  - "오행의 균형을 살피는 중..."
  - "오늘의 기운을 읽는 중..."
  - "AI 도사에게 연락하는 중..."
  - "운세를 정리하는 중..."

### type=report (심층 리포트)
- **진입**: `/result`에서 "심층 리포트 보기" 클릭
- **API**: `POST /api/report`
- **완료 후 이동**: `/report`
- **문구**:
  - "사주 원국을 깊이 분석하는 중..."
  - "오행의 상생상극을 풀어보는 중..."
  - "올해의 대운 흐름을 살피는 중..."
  - "맞춤 행동 조언을 준비하는 중..."
  - "심층 리포트를 정리하는 중..."

## 동작 흐름

1. 쿼리 파라미터 `type` 확인 (없으면 `/input`으로 리다이렉트)
2. sessionStorage에서 필요한 데이터 로드
3. 로딩 문구 순환 시작 (2초 간격)
4. 해당 API 호출 (saju 타입은 사주 계산 + AI 해석 순차 호출)
5. 결과를 sessionStorage에 캐싱 (`sajuResult`, `sajuInterpretation`, `sajuReport`)
6. 최소 3초 표시 후 목적지 페이지로 이동

## 에러 처리
- API 실패 시 에러 메시지 표시 + "다시 시도" 버튼
- "다시 시도" → 같은 API 재호출
- "돌아가기" → 이전 페이지로 이동

## 이동 경로

| 요소 | 이동 대상 |
|------|-----------|
| saju 완료 | `/result` |
| report 완료 | `/report` |
| 에러 시 돌아가기 | `router.back()` |
