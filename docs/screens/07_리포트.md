# 07. 심층 리포트 페이지

> 경로: `/report` | 파일: `src/app/report/page.tsx` | 렌더링: Client Component

## 역할

유료 심층 리포트를 표시하는 페이지. `/result`에서 "심층 리포트 보기" 버튼을 통해 진입합니다.

## 진입 조건

- sessionStorage에 `sajuResult`와 `sajuInput`이 필요
- 없으면 `/input`으로 리다이렉트

## 동작 흐름

1. sessionStorage에서 사주 결과 + 입력 데이터 읽기
2. `POST /api/report` → 심층 리포트 수신
3. 섹션별 카드로 표시

## 화면 구성

### 1) 페이지 제목
- ReportIcon (문서 아이콘) + "심층 리포트" (lg, bold)

### 2) 섹션 1: 사주 개요
- **라벨**: CompassIcon + "사주 개요"
- **내용 3개**:
  - "나의 일간": 일간에 대한 쉬운 설명
  - "오행 분석": 오행 분포 해석
  - "성격과 성향": 성격/성향 해석
- 각 항목은 border-t로 구분

### 3) 섹션 2: 오늘의 상세 해석
- **라벨**: CalendarIcon + "오늘의 상세 해석"
- **내용 4개**:
  - "전체 흐름": 오늘의 전체 흐름
  - HeartIcon + "연애운": 연애운 상세 해석
  - BriefcaseIcon + "직장/일운": 직장운 상세 해석
  - CoinIcon + "재물운": 재물운 상세 해석
- 아이콘 색상: 하트=pink, 서류가방=blue, 동전=yellow

### 4) 섹션 3: 구체적 행동 조언
- **라벨**: LightbulbIcon + "구체적 행동 조언"
- **3가지 조언**: 번호 뱃지(6x6 원형, bg-foreground) + 텍스트

### 5) 섹션 4: 이번 주 흐름
- **라벨**: TrendIcon (심전도 모양) + "이번 주 흐름"
- **내용**: 이번 주 흐름 미리보기 텍스트

### 6) 섹션 5: 마무리 메시지
- 검은 배경(bg-foreground), 흰 텍스트
- 마무리 한마디 (sm, medium)

### 7) 하단 버튼
- "결과 화면으로 돌아가기" → `/result` (outline, h-11)

## 로딩 화면
- 스피너 + "심층 리포트를 작성하고 있습니다..." (animate-pulse)

## 에러 화면
- "오류가 발생했습니다" + 에러 메시지 + "결과 화면으로 돌아가기" 버튼

## 캐시 정책
- 리포트 결과를 `sessionStorage`(`sajuReport`)에 캐싱
- 재진입 시 캐시 있으면 API 호출 스킵 → 즉시 표시
- `/input`에서 새로 제출하면 캐시 삭제 → 새로 생성

## API 호출
- `POST /api/report`
- Body: analysis (fourPillars, fiveElements, dayMaster, dayMasterElement, dayMasterStrength), daily, gender
- Response: `{ report: ReportData }`
