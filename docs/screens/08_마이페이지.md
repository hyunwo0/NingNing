# 08. 마이페이지

> 경로: `/my` | 파일: `src/app/my/page.tsx` | 렌더링: Client Component

## 역할

로그인한 사용자의 정보와 저장된 운세 결과를 보여주는 페이지입니다.

## 인증 요구

- **로그인 필수**: 미로그인 시 `/login`으로 리다이렉트
- `useAuth()` 훅으로 인증 상태 확인

## 화면 구성

### 1) 뒤로가기 링크
- "← 홈으로" → `/`

### 2) 사용자 정보 영역
- **카드**: 라운드 카드
- **아바타**: 12x12 원형, 사용자 아이콘 (SVG)
- **이름**: `user_metadata.full_name` 또는 `user_metadata.name` 또는 "회원"
- **이메일**: `user.email` (truncate)

### 3) 저장된 운세 목록
- **제목**: "저장된 운세 (N)" — 저장 개수 표시
- **빈 상태**: 북마크 아이콘 + "저장된 운세가 없습니다" + "운세 보러 가기 →" 링크
- **목록 있을 때**: 각 결과 카드
  - **날짜** + **삭제 버튼** (hover 시 나타남, 빨간색 전환)
  - **총평** (sm, medium)
  - **키워드 태그** (rounded-full, bg-muted)
  - **3축 점수**: ScoreBadge (연애/일/재물)
    - 점수별 색상: 7+ 초록, 4+ 노랑, 그 외 빨강

### 4) 하단 버튼들
- **로그아웃 버튼**: h-11, 테두리, "로그아웃"
  - 동작: `signOut()` → `/`로 이동
- **계정 삭제 링크**: 작은 텍스트, hover 시 빨간색
  - 클릭 시 모달 오픈

### 5) 계정 삭제 안내 모달
- 반투명 오버레이 (bg-black/50)
- 모달 카드 (max-w-sm)
  - "계정 삭제 안내" 제목
  - "계정 삭제를 원하시면 아래 이메일로 문의해주세요."
  - **이메일**: support@ningning.app
  - "확인" 버튼으로 닫기

## 데이터
- **저장된 결과**: localStorage `savedResults`에서 로드
  - 최신 순 정렬
  - 각 항목: id, date, dailySummary, keywords, scores (love/work/money)
- **삭제**: localStorage에서 해당 id 제거 후 업데이트

## 로딩 화면
- 스피너 (인증 확인 중 또는 미로그인 상태)
