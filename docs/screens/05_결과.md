# 05. 운세 결과 페이지

> 경로: `/result` | 파일: `src/app/result/page.tsx` | 렌더링: Client Component

## 역할

입력된 사주 데이터를 기반으로 사주 계산 → AI 해석 → 결과를 표시하는 핵심 페이지입니다.

## 동작 흐름

1. sessionStorage에서 `sajuResult` 읽기 (없으면 `/input`으로 리다이렉트)
2. 캐시된 사주 결과 + AI 해석을 즉시 표시
3. AI 해석이 캐시에 없으면 재시도 배너 표시 (재시도 버튼으로 `/api/interpret` 호출)

> 모든 API 호출(`/api/saju` + `/api/interpret`)은 `/loading-screen`에서 수행. 이 페이지는 캐시 읽기 + 재시도만 담당

## 화면 구성

### 1) 헤더 영역
- **좌측**: "홈으로" 링크 → `/`
- **우측**: 결과 저장 버튼 (AI 해석 완료 시만 표시)
  - 미저장: 빈 북마크 아이콘 + "결과 저장"
  - 저장됨: 채워진 북마크 아이콘 + "저장됨"

### 2) AI 해석 실패 배너 (조건부)
- 빨간 배경, "AI 해석을 불러오지 못했습니다" + "다시 시도" 버튼

### 3) 오늘의 무드
- **카드**: 검은 배경(bg-foreground), 흰 텍스트
  - "오늘의 무드" 라벨
  - 한글 무드명 (xl, bold)
  - 무드 설명 한 줄
  - 한글 키워드 태그들

### 4) 3축 운세 카드 (EnergyCard x3)
- **연애** (HeartIcon), **일/직장** (BriefcaseIcon), **재물** (CoinIcon)
- 각 카드 구성:
  - 헤더: 아이콘 + 축 이름(한글) + 상태명 태그(한글)
  - 점수 바: 1.5px 높이, 점수에 따라 초록/노랑/빨강
  - **tip (메인)**: 핵심 한 줄 행동 안내 (bold)
  - **interpretation (서브)**: 상세 해설 (muted)

### 5) 오늘의 전략
- **카드**: 검은 배경, 흰 텍스트, 중앙 정렬
  - "오늘의 전략" 라벨
  - 한국어 카피 (lg, bold, 메인)
  - 영어 카피 (sm, 서브)

### 6) 행운 부스터
- **제목**: "행운 부스터" (base, semibold)
- **4개 항목** (가로 2열: 제목 | 값):
  - 스타일 코드 / 행운 숫자 / 에너지 방향 / 골든 타임
  - 제목: muted, 고정 너비
  - 값 메인: bold
  - 값 서브: muted, 작은 글씨
  - " — " 또는 " - " 기준으로 메인/서브 자동 분리

### 7) 사주 해석 (접이식)
- `<details>` 태그, "사주 해석 보기" + 화살표
- 에너지 특성 한 줄 + 해석 2~3문장

### 8) 하단 버튼들
| 순서 | 텍스트 | 스타일 | 이동 |
|------|--------|--------|------|
| 1 | AI에게 추가 질문하기 | primary, h-12 | `/question` |
| 2 | 심층 리포트 보기 | secondary, h-11 | `/loading-screen?type=report` 또는 `/report` |
| 3 | 공유하기 | outline, h-11 | ShareModal 열기 |
| 4 | 다른 사주로 보기 | outline, h-11 | `/input` |

### 9) 공유 모달 (ShareModal)
- AI 해석 완료 시에만 "공유하기" 버튼 노출
- 모바일: "결과 공유" (OS 공유 시트 + 홈 URL)
- PC: "결과 저장" (이미지 다운로드)

## 에러 화면
- "오류가 발생했습니다" + 에러 메시지 + "다시 입력하기" 버튼

## 데이터 저장
- 결과 저장 → `localStorage.setItem('savedResults', ...)` (오늘 날짜 기준 중복 방지)
