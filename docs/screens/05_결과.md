# 05. 운세 결과 페이지

> 경로: `/result` | 파일: `src/app/result/page.tsx` | 렌더링: Client Component

## 역할

입력된 사주 데이터를 기반으로 사주 계산 → AI 해석 → 결과를 표시하는 핵심 페이지입니다.

## 동작 흐름

1. sessionStorage에서 `sajuInput` 읽기 (없으면 `/input`으로 리다이렉트)
2. **캐시 확인**: `sajuResult`가 있고 `sajuNeedsFetch`가 없으면 캐시 사용 → API 호출 스킵
3. 캐시 없거나 새 입력(`sajuNeedsFetch=true`)이면:
   - `POST /api/saju` → 사주 계산 결과 수신 → 즉시 화면 표시
   - `POST /api/interpret` → AI 해석 수신 (백그라운드) → 스켈레톤 → 실제 내용으로 교체
   - 결과를 sessionStorage에 캐싱 (`sajuResult`, `sajuInterpretation`)

### 캐시 정책

| 진입 방식 | 동작 |
|-----------|------|
| `/input`에서 새로 제출 | `sajuNeedsFetch=true` 설정 → API 호출 |
| `/question`, `/report`에서 뒤로가기 | 캐시 사용 → API 호출 없음 |
| 주소창 직접 입력으로 재진입 | 캐시 있으면 사용, 없으면 `/input`으로 리다이렉트 |
| "다른 사주로 보기" 버튼 | `/input` → 새 제출 → API 호출 |

## 화면 구성

### 1) 헤더 영역
- **좌측**: "홈으로" 링크 → `/`
- **우측**: 결과 저장 버튼 (AI 해석 완료 시만 표시)
  - 미저장: 빈 북마크 아이콘 + "결과 저장"
  - 저장됨: 채워진 북마크 아이콘 + "저장됨"

### 2) AI 해석 실패 배너 (조건부)
- 빨간 배경, "AI 해석을 불러오지 못했습니다" + "다시 시도" 버튼

### 3) 사주 팔자 표시
- **카드**: 라운드 카드, "나의 사주 팔자" 라벨
- **4기둥 표시**: 년주 | 월주 | 일주 | 시주 (grid 4열)
  - 각 기둥: 라벨(10px) + 천간(xl bold) + 지지(lg)
  - 시주 없으면 "미상" 표시
- **하단 정보**: 일간 + 오행 | 신강/신약/중화

### 4) 오행 분포
- **카드**: "오행 분포" 라벨
- **막대 차트**: 목/화/토/금/수 (5열)
  - 색상: 초록/빨강/노랑/회색/파랑
  - 높이: 최대값 대비 비율, 최소 8%
  - 상단에 숫자, 하단에 오행 이름
- **부족한 오행**: "부족한 오행: OO" (있을 때만)

### 5) AI 한 줄 총평
- **카드**: 검은 배경(bg-foreground), 흰 텍스트
  - 총평 문장 (lg, semibold)
  - 키워드 태그들 (rounded-full, bg-background/20)
- **로딩 중**: 스켈레톤 UI

### 6) 3축 운세 카드 (ReadingCard x3)
- **연애** (HeartIcon), **일/직장** (BriefcaseIcon), **재물** (CoinIcon)
- 각 카드 구성:
  - 헤더: 아이콘 + 축 이름 + 키워드 태그
  - 점수 바: 1.5px 높이, 점수에 따라 초록/노랑/빨강
  - 해석 텍스트: AI 해석 우선, 없으면 엔진 요약
  - 행동 조언: AI 조언 (있을 때)

### 7) 오늘의 행동 조언
- **상단**: "오늘 하면 좋은 것" (초록 라벨) + 내용
- **하단**: "오늘 피하면 좋은 것" (빨강 라벨) + 내용
- **로딩 중**: 스켈레톤 UI

### 8) 행운 단서
- "오늘의 행운 단서" 라벨
- 태그 형태 (rounded-full, bg-muted)

### 9) 사주 근거 (접이식)
- `<details>` 태그 사용, "사주 근거 보기" + 화살표
- 열면 사주 배경 설명 텍스트

### 10) 하단 버튼들
| 순서 | 텍스트 | 스타일 | 이동 |
|------|--------|--------|------|
| 1 | AI에게 추가 질문하기 | primary, h-12 | `/question` |
| 2 | 심층 리포트 보기 | secondary, h-11 | `/report` |
| 3 | 공유하기 | outline, h-11 | ShareModal 열기 |
| 4 | 다른 사주로 보기 | outline, h-11 | `/input` |

### 11) 공유 모달 (ShareModal)
- AI 해석 완료 시에만 "공유하기" 버튼 노출
- ShareModal 컴포넌트로 총평, 3축 점수, 행운 단서 전달

## 로딩 화면
- 스피너 + "사주를 분석하고 있습니다..." (animate-pulse)

## 에러 화면
- "오류가 발생했습니다" + 에러 메시지 + "다시 입력하기" 버튼

## 데이터 저장
- 사주 계산 결과 → `sessionStorage.setItem('sajuResult', ...)`
- 결과 저장 → `localStorage.setItem('savedResults', ...)` (오늘 날짜 기준 중복 방지)

## 이동 경로

| 요소 | 이동 대상 |
|------|-----------|
| 홈으로 | `/` |
| AI 추가 질문하기 | `/question` |
| 심층 리포트 보기 | `/report` |
| 다른 사주로 보기 | `/input` |
| 에러 시 다시 입력 | `/input` |
