# 02. 테마별 랜딩 페이지

> 경로: `/landing/[theme]` | 파일: `src/app/landing/[theme]/page.tsx` | 렌더링: Server Component

## 역할

특정 주제(연애운, 이직운)에 맞춘 랜딩 페이지. 동적 라우트로 여러 테마를 하나의 파일로 관리합니다.

## 지원 테마

| 경로 | 테마 | 아이콘 |
|------|------|--------|
| `/landing/love` | 연애운 | HeartLargeIcon (채워진 하트) |
| `/landing/career` | 이직운 | BriefcaseIcon (서류가방) |

- 유효하지 않은 테마 → `/`로 리다이렉트
- `generateStaticParams()`로 빌드 시 정적 생성

## 화면 구성

### 1) 히어로 영역
- **테마 아이콘**: 16x16 검은 사각형 안에 테마별 큰 아이콘
- **제목**: 테마별 타이틀 (예: "오늘의 연애운")
- **서브타이틀**: 한 줄 후킹 문구
- **설명**: 상세 설명 문구
- **CTA 버튼**: 테마별 CTA 텍스트 → `/input`

### 2) 샘플 결과 미리보기
- **제목**: "이런 결과를 받아볼 수 있어요"
- **카드**: 라운드 카드 안에 NingNing AI 프로필 + 샘플 운세 텍스트
  - NingNing 로고 (6x6 원형) + "NingNing AI" 라벨

### 3) 특징 소개 (3개)
- 테마별 다른 아이콘 세트
  - love: HeartIcon, SparklesIcon, ChatIcon
  - career: TrendingIcon, ClockIcon, ChatIcon
- 레이아웃: 메인 페이지와 동일 (아이콘 + 텍스트 가로 배치)

### 4) 하단 CTA + 돌아가기
- **CTA 버튼**: 테마별 CTA 텍스트 → `/input` (w-full)
- **돌아가기 링크**: "← 메인으로 돌아가기" → `/`

### 5) 푸터
- "NingNing · AI 운세 서비스"
- 면책 문구

## 테마별 콘텐츠

### love (연애운)
- 타이틀: "오늘의 연애운"
- 서브타이틀: "그 사람과의 타이밍, 사주로 읽어드립니다"
- CTA: "내 연애운 확인하기"
- 특징: 연애 타이밍 분석 / 상대방과의 궁합 힌트 / AI 맞춤 연애 조언

### career (이직운)
- 타이틀: "오늘의 이직운"
- 서브타이틀: "지금 움직여도 될까? 사주가 알려주는 타이밍"
- CTA: "내 직장운 확인하기"
- 특징: 직장운 흐름 분석 / 변화 타이밍 판단 / AI 맞춤 커리어 조언
