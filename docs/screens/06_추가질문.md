# 06. AI 추가질문 페이지

> 경로: `/question` | 파일: `src/app/question/page.tsx` | 렌더링: Client Component

## 역할

운세 결과를 본 사용자가 개인 상황에 맞는 추가 질문을 할 수 있는 채팅 페이지입니다. 사주 맥락을 기반으로 AI가 맞춤 답변을 생성합니다.

## 진입 조건

- sessionStorage에 `sajuResult`와 `sajuInput`이 필요
- 없으면 `/input`으로 리다이렉트

## 무료/유료 규칙

- **무료 질문**: 2회
- **이후**: 유료 (현재 결제 미구현, 2회 제한만 적용)

## 화면 구성

### 1) 헤더 (고정)
- **좌측**: 뒤로가기 아이콘 (ChevronLeft) → `router.back()`
- **중앙**: "AI 추가질문" (sm, semibold)
- **우측**: 남은 횟수 표시
  - "2회 남음" / "1회 남음" / "무료 소진"

### 2) 대화 영역 (스크롤)
#### 초기 상태 (질문 0개)
- **안내 카드**: "오늘의 사주 흐름을 바탕으로 궁금한 점을 물어보세요..."
- **예시 질문 4개** (버튼 형태, 클릭 시 입력창에 삽입):
  - "이직을 준비 중인데 지금 움직여도 될까요?"
  - "최근 만난 사람과 잘 될 수 있을까요?"
  - "이번 달 큰 지출을 해도 괜찮을까요?"
  - "시험을 앞두고 있는데 결과가 어떨까요?"

#### 대화 진행 중
- **사용자 질문** (오른쪽 정렬): 검은 배경 말풍선, rounded-br-md
- **AI 답변** (왼쪽 정렬): 흰 배경 + 테두리 말풍선, rounded-bl-md
- **로딩 표시**: 3개 점 바운스 애니메이션
- **에러 표시**: 빨간 배경 + 에러 메시지 + "다시 시도" 버튼

### 3) 입력 영역 (하단 고정)
#### 질문 가능 상태
- **텍스트 입력**: textarea (rows=1, 자동 리사이즈 없음)
  - placeholder: "궁금한 점을 물어보세요..."
  - Enter로 전송 (Shift+Enter는 줄바꿈)
- **전송 버튼**: SendIcon, rounded-xl

#### 무료 소진 상태
- "무료 질문 2회를 모두 사용했습니다"
- "결과 화면으로 돌아가기" 버튼 → `/result`

## API 호출
- `POST /api/question`
- Body: analysis, daily, gender, question, previousQA
- Response: `{ answer: string }`

## 상태 관리
- `sajuData`: 사주 분석 데이터 (sessionStorage에서 로드)
- `qaHistory`: QAItem[] (질문-답변 이력)
- `inputText`: 현재 입력 텍스트
- `isLoading`: 답변 생성 중
- `error`: 에러 메시지

## 캐시 정책
- 대화 이력(`qaHistory`)을 `sessionStorage`(`sajuQAHistory`)에 저장/복원
- 페이지 재진입 시 이전 대화 + 사용 횟수가 유지됨
- `/input`에서 새로 제출하면 대화 이력 초기화

## 에러 처리
- 네트워크 오류: "네트워크 연결을 확인해주세요."
- API 오류: 서버 에러 메시지 표시
- 실패 시 질문 텍스트를 입력창에 복원
