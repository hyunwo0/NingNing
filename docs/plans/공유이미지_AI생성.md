# 공유 이미지 AI 생성 계획

> 작성일: 2026-04-12

## 목표

운세 결과를 공유할 때, 텍스트 나열 대신 AI가 생성한 감성 일러스트 + 텍스트 오버레이로 시각적으로 매력적인 공유 이미지를 만든다.

## 현재 문제

- 공유 이미지가 Canvas API로 텍스트만 나열 → 밋밋함
- 공유받은 사람 입장에서 "나도 해볼까?" 동기 부여가 약함
- 바이럴 효과가 낮음

## 해결 방향

```
운세 결과 데이터 (행동 조언, 전략, 무드)
  → AI 이미지 프롬프트 생성 (사람 + 행동 중심)
  → Gemini(나노바나나) API로 일러스트 생성
  → 결과 화면 상단에 이미지 표시 (텍스트 오버레이 없음)
  → 공유 시에도 같은 이미지 활용
```

## 기술 스택

- **이미지 생성**: Google Gemini API (나노바나나)
- **모델**: `gemini-2.5-flash-image`
- **SDK**: `@google/genai`
- **API 키**: Google AI Studio에서 발급 → `GOOGLE_AI_API_KEY`

## 핵심 원칙

1. **이미지 영역 분리**: 텍스트 오버레이 없이 이미지만 표시
2. **결과 화면에서 노출**: 공유 시가 아니라 결과 보자마자 보여야 함
3. **사람 + 행동 중심**: 행동 강령을 시각화 (사람이 그 행위를 하는/피하는 장면)

## 구현 흐름

### 1) 로딩 단계에서 이미지 생성
```
/loading-screen?type=saju
  ├─ /api/saju (사주 계산)
  ├─ /api/interpret (AI 해석) ── 병렬 ── /api/share-image (이미지 생성)
  └─ 모두 완료 → sessionStorage 캐싱 → /result
```

### 2) 결과 화면에서 표시
```
┌─────────────────────────┐
│ ←     오늘의 운세   🏠👤 │ ← GNB
│                          │
│ ┌──────────────────────┐ │
│ │                       │ │
│ │   AI 생성 일러스트     │ │ ← 이미지 영역 (텍스트 없음)
│ │   (사람 + 행동 장면)  │ │
│ │                       │ │
│ └──────────────────────┘ │
│                          │
│ 오늘의 무드              │ ← 기존 결과 UI
│ 3축 운세 카드            │
│ ...                      │
└─────────────────────────┘
```

### 3) 이미지 프롬프트 규칙 (행동 중심)
- 전략/조언을 시각화 (예: "보여주기보다 쌓는 날" → 조용히 책을 읽는 사람)
- 일러스트 스타일 (포토리얼 아님)
- K-감성 아트 스타일
- 텍스트/글자 포함하지 않기
- 1:1 비율

## 비용 예상
