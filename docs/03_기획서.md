# 서비스 기획서 — NingNing AI 운세 서비스

---

## 1. 서비스 컨셉

### 1.1 한 줄 정의
**"내 사주를, 오늘의 선택 언어로 풀어주는 AI 운세"**

### 1.2 컨셉 키워드
- **정통**: 만세력·오행·사주 구조 기반 계산
- **현실**: 생활 언어로 풀어쓴 해석
- **실용**: 오늘 바로 적용할 수 있는 행동 조언
- **신뢰**: 계산 근거를 보여주는 투명함

### 1.3 브랜드 톤
| 원칙 | 설명 |
|------|------|
| 정통하지만 무겁지 않게 | 사주 용어를 쓰되 설명을 곁들인다 |
| 따뜻하지만 과장하지 않게 | 공감하되 허위 희망은 주지 않는다 |
| 담백하지만 한 줄은 강하게 | 전체는 차분하되 핵심 문장은 기억에 남게 |

> 추천 느낌: **믿을 만하고, 쉽게 읽히고, 바로 적용된다.**

### 1.4 하지 않을 것
- 첫 버전에 타로·관상·전문가 매칭 동시 출시
- 점수 놀이 중심 UX
- 무제한 AI 채팅
- 공포를 유도하는 과장형 메시지
- 확정적 미래 예언

### 1.5 적극 허용할 것
- 사주 오행 기반 행운 숫자/색상/방위 제시
- 도박/게임 승부운 질문에 대한 사주 맥락 답변 (면책 문구 포함)
- 연애/고백/이별 등 관계 관련 구체적 사주 해석
- 홀덤 행운 핸드, 로또 번호 등 재미 요소 (오행 근거 기반)

## 2. 사용자 정의

### 2.1 핵심 페르소나

#### 페르소나 A: "이직 고민 직장인" (28세, 여성)
- 현재 직장 2년차, 이직을 고민 중
- 운세 앱을 가끔 보지만 "재미로만" 본다고 말함
- 실제로는 "지금 움직여도 되나?"에 대한 심리적 확인을 원함
- 기대: 구체적이고 현실적인 해석, 애매하지 않은 행동 조언

#### 페르소나 B: "연애 상담 필요한 대학생" (24세, 여성)
- 썸 단계에서 상대방 마음이 궁금
- 친구들과 운세 결과 공유하며 재미를 느낌
- 기대: 가볍게 시작하되 해석이 깊으면 유료도 고려

#### 페르소나 C: "새해 운세 검색자" (35세, 남성)
- 매년 신년에 "2026 운세" 검색
- 사주를 깊이 믿진 않지만 구조적 설명엔 신뢰 느낌
- 기대: 논리적이고 체계적인 해석, 감성적 과장은 거부감

### 2.2 사용자 맥락
| 맥락 | 특성 |
|------|------|
| 이용 시간 | 출근 전 아침, 점심시간, 취침 전 |
| 이용 환경 | 모바일 웹 (검색 유입) |
| 이용 빈도 | 고민 있을 때 집중 → 습관화 시 매일 |
| 결제 심리 | "무료가 좋았으면 유료는 더 좋겠지" → 단건 결제 |

## 3. 사용자 여정 설계

### 3.1 핵심 흐름

```
[1. 랜딩] → [2. 사주 입력] → [3. 첫 결과] → [4. AI 추가질문] → [5. 결제·저장]
```

#### Step 1: 랜딩
- **목표**: 기대치 형성, 입력 유도
- **핵심 요소**:
  - 메인 헤드라인 + 서브카피
  - 샘플 결과 문장 1~2개 (실제 톤 미리보기)
  - CTA: "3분 안에 오늘의 흐름 보기"
- **전환 기준**: CTA 클릭 → 사주 입력 진입

#### Step 2: 사주 입력
- **목표**: 빠르고 정확한 정보 수집
- **핵심 요소**:
  - 양/음력 토글 (기본: 양력)
  - 생년월일 선택 (드롭다운/날짜 선택기)
  - 출생시간 선택 (12시진 + "모름")
  - 성별 선택
  - 정확도 안내 문구: "출생시간을 알면 더 정확한 해석을 받을 수 있어요"
- **전환 기준**: 입력 완료 → 결과 생성 시작
- **설계 원칙**: 필수 항목 최소화, 입력 과정 2분 이내

#### Step 3: 첫 결과 화면 (핵심 화면)
- **목표**: "맞는 느낌" 제공, 더 알고 싶은 욕구 생성
- **핵심 요소**:
  - 한 줄 총평 (가장 먼저, 가장 크게)
  - 오늘의 키워드 1~2개
  - 3축 카드 (연애/일/재물)
  - 오늘 하라 / 피하라
  - 행운 단서
  - CTA 1: "지금 상황을 붙여 더 물어보기" (AI 질문)
  - CTA 2: "내 사주 흐름 자세히 보기" (유료 리포트)
- **전환 기준**: AI 질문 시작 또는 유료 CTA 클릭
- **설계 원칙**: 이 화면이 서비스의 전부라고 생각하고 만든다

#### Step 4: AI 추가질문
- **목표**: 개인화된 깊은 해석 경험 제공, 결제 동기 강화
- **핵심 요소**:
  - 질문 입력 필드 (플레이스홀더: "이직 준비 중인데 지금 움직여도 될까요?")
  - AI 답변 (사주 맥락 유지, 현재 상황 반영)
  - 후속 질문 유도 ("더 궁금한 게 있으신가요?")
  - 남은 질문 횟수 표시
  - 질문 소진 시: 질문 팩 결제 유도
- **전환 기준**: 결제 또는 저장

#### Step 5: 결제·저장
- **목표**: 결제 완료, 재방문 장치 설치
- **핵심 요소**:
  - 심층 리포트 상품 설명 (무료와의 차이 명확히)
  - 가격 표시 + 결제 버튼
  - 결제 완료 후: 리포트 즉시 열람 + 저장
  - 재방문 유도: "내일의 흐름도 확인해보세요"
  - 알림 옵트인 (선택)

### 3.2 반복 루프

#### 매일 반복 루프
```
오늘의 총평 확인 → 추가 질문 1회 → 저장 → 저녁 다시 보기
```

#### 상황 기반 루프
```
테마 랜딩(연애/이직/재물) → 결과 → 리포트 결제
```

#### 시즌성 루프
```
신년운세 → 이직 시즌 → 궁합 시즌 → 재유입
```

## 5. 핵심 화면 상세 기획

### 5.1 결과 화면 (Result View)

#### 레이아웃 구성 (위에서 아래 순서)

```
┌─────────────────────────────┐
│  오늘의 날짜 + 사용자 이름   │
├─────────────────────────────┤
│                             │
│  ★ 오늘의 한 줄 총평 ★      │
│  (가장 크고 강한 서체)       │
│                             │
├─────────────────────────────┤
│  오늘의 키워드: #정리 #집중   │
├─────────────────────────────┤
│                             │
│  [연애] 카드    80~150자     │
│  행동 조언 1줄               │
│                             │
│  [일/직장] 카드  80~150자    │
│  행동 조언 1줄               │
│                             │
│  [재물] 카드    80~150자     │
│  행동 조언 1줄               │
│                             │
├─────────────────────────────┤
│  ✓ 오늘 하면 좋은 것        │
│  ✗ 오늘 피하면 좋은 것       │
├─────────────────────────────┤
│  🍀 행운 단서 1~2개         │
├─────────────────────────────┤
│                             │
│  [지금 상황을 붙여 더 물어보기] ← AI 질문 CTA
│                             │
│  [내 사주 흐름 자세히 보기]   ← 유료 리포트 CTA
│                             │
├─────────────────────────────┤
│  결과 저장 | 공유하기         │
└─────────────────────────────┘
```

#### 설계 원칙
- 한 줄 총평은 스크롤 없이 바로 보여야 한다
- 3축 카드는 좌우 스와이프 또는 수직 스크롤로 탐색
- CTA는 해석을 다 읽은 후 자연스러운 위치에 배치
- 무료 영역과 유료 영역의 경계가 명확해야 함

### 5.2 사주 입력 화면

#### 입력 순서
1. 양력/음력 선택 (토글, 기본값: 양력)
2. 생년월일 (연/월/일 드롭다운 또는 날짜 선택기)
3. 출생시간 (12시진 선택 + "모름" 버튼)
4. 성별 (남/여 버튼)
5. 결과 보기 버튼

#### UX 세부 사항
- 출생시간 "모름" 선택 시: "시간 없이도 해석이 가능해요. 다만 출생시간을 알면 더 정밀한 결과를 받을 수 있어요." 안내
- 음력 선택 시: 윤달 여부 체크박스 표시
- 프로그레스바 또는 단계 표시 (1/2, 2/2 등)
- 입력 완료 후 로딩: "사주를 분석하고 있어요..." 애니메이션

## 6. 콘텐츠 전략

### 6.1 AI 해설 파이프라인

```
[입력 정규화] → [사주 계산] → [해석 템플릿] → [문장 생성] → [가드레일]
```

| 단계 | 역할 | 담당 |
|------|------|------|
| 입력 정규화 | 양/음력 변환, 시간 미상 처리 | 시스템 |
| 사주 계산 | 오행·일간·기본 구조 산출 | 사주 엔진 |
| 해석 템플릿 | 주제별 해석 포인트 추출 | 프롬프트 |
| 문장 생성 | 현실 언어로 자연스럽게 변환 | AI (LLM) |
| 가드레일 | 과장·공포·금지 주제 필터링 | 시스템 |

### 6.2 톤앤매너 가이드라인

#### 좋은 예시
> "오늘은 무언가를 크게 벌이기보다 정리하고 방향을 좁히는 선택에 강점이 있는 날입니다. 연애에서는 애매한 표현보다 솔직한 한 문장이 유리하고, 일에서는 새로운 시작보다 기존 흐름을 정리하는 쪽이 성과를 만듭니다."

#### 나쁜 예시
> "오늘 매우 위험한 날입니다!!! 절대로 큰 결정을 하지 마세요. 연애운이 최악이니 고백은 무조건 피하세요."

#### 톤 원칙
| DO | DON'T |
|-----|--------|
| "~에 강점이 있는 날입니다" | "~해야 합니다" |
| "~하는 쪽이 유리합니다" | "~하면 큰일 납니다" |
| "~을 고려해보세요" | "반드시 ~하세요" |
| "흐름상 ~에 집중하면 좋겠습니다" | "운이 최악입니다" |

### 6.3 카피 가이드

| 위치 | 카피 |
|------|------|
| 메인 헤드라인 | 내 사주를, 오늘의 선택에 맞게 풀어주는 AI 운세 |
| 서브카피 | 정통 사주 구조를 바탕으로 연애, 일, 재물 흐름을 현실 언어로 해설합니다 |
| 온보딩 CTA | 3분 안에 오늘의 흐름 보기 |
| 유료 CTA | 내 사주 흐름 자세히 보기 |
| 질문 CTA | 지금 상황을 붙여 더 물어보기 |

## 7. 유료 전환 설계

### 7.1 전환 지점

```
무료 결과 화면
  ↓ (충분한 가치 전달)
AI 추가질문 (무료 1~3회)
  ↓ (더 깊은 해석 경험)
질문 소진 → 질문 팩 결제 유도
  또는
결과 하단 → 심층 리포트 결제 유도
```

### 7.2 전환 원칙
- 무료 영역에서 충분한 가치를 느껴야 유료 전환이 자연스러움
- "왜 사야 하는지"가 가격보다 먼저 전달되어야 함
- 결제 전 리포트 미리보기(일부 잠금)로 기대치 형성
- 결제 후 즉시 콘텐츠 열람 가능

### 7.3 실험 계획

| 실험 | 변수 | 측정 지표 |
|------|------|----------|
| 가격 × 분량 | 리포트 가격(3,900/5,900/7,900) × 분량 | 결제율, 만족도 |
| 무료 질문 횟수 | 1회 / 3회 / 5회 | 전환율, 질문 진입률 |
| 테마 랜딩 | 연애 / 이직 / 재물 | 결과 도달율, 결제율 |

## 8. 재방문 설계

### 8.1 일상 재방문 장치
- 오늘의 총평이 매일 바뀜 (같은 사주라도 일진이 다르므로)
- 무료 AI 질문 횟수 일 단위 초기화
- 저장된 결과 재열람 유도
- "어제의 조언이 도움이 되었나요?" 후속 경험

### 8.2 시즌 재유입 장치
- 신년운세 (12~1월)
- 이직 시즌 (1~3월, 8~9월)
- 발렌타인/화이트데이 궁합 (2~3월)
- 대입/시험 시즌 (11월)
