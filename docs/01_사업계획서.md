# 사업계획서 — NingNing (닝닝) AI 운세 서비스

---

## 1. 사업 개요

### 1.1 서비스명
**NingNing (닝닝)** — 내 사주를, 오늘의 선택에 맞게 풀어주는 AI 운세

### 1.2 사업 비전
정통 사주 계산을 기반으로, AI가 현실 언어로 해석해주는 한국형 운세 서비스를 만든다. 사용자가 첫 결과 화면에서 신뢰를 느끼고, 자연스럽게 유료 전환까지 이어지는 경험을 설계한다.

### 1.3 미션
"점수보다 해석, 기능보다 경험, 예언보다 적용 가능한 조언"
- 사주의 정통성은 유지하되, 난해함은 낮춘다
- AI는 점쟁이가 아니라 해설사 역할을 한다
- 사용자가 '지금 써먹을 수 있는' 해석을 제공한다

### 1.4 핵심 가치 제안
| 구분 | 기존 운세 서비스 | NingNing |
|------|-----------------|----------|
| 결과 형태 | 점수, 별점, 짧은 한 줄 | 맥락 있는 해석 문장 + 행동 조언 |
| 신뢰 근거 | 불명확 | 정통 사주 계산(만세력·오행·일간) 기반 |
| 결제 유도 | 반복 팝업, 구독 강요 | 결과 → 질문 → 결제 자연 흐름 |
| AI 활용 | 단순 생성 | 사주 계산 결과를 현실 언어로 변환 |
| 톤앤매너 | 감성적/공포 유발 | 담백하고 실용적 |

## 2. 시장 분석

### 2.1 시장 규모
- 한국 운세/사주 시장: 연간 약 3,000~5,000억 원 규모 (오프라인 포함)
- 모바일/온라인 운세 시장: 연간 약 500~1,000억 원 규모로 지속 성장
- AI 기반 개인화 콘텐츠 시장 확대 추세

### 2.2 타깃 사용자
| 구분 | 1차 타깃 | 2차 타깃 |
|------|----------|----------|
| 연령 | 25~39세 | 20~24세, 40~49세 |
| 성별 | 여성 중심 (70%) | 남성 확대 |
| 행동 | 운세 앱 경험 있음, 사주에 관심 | 가벼운 호기심, 공유 목적 |
| 니즈 | 연애·이직·재물 관련 구체적 해석 | 오늘의 운세, 재미 |
| 채널 | 모바일 웹 검색 유입 | SNS 공유 유입 |

### 2.3 시장 기회
- 기존 운세 서비스들은 기능은 넓지만 첫 방문 가치가 흐려지기 쉬움
- 한국 사용자는 별자리 감성보다 사주·만세력·오행 같은 구조적 설명에 익숙함
- "AI가 점을 친다"보다 "사주를 쉽게 해석해준다"는 메시지가 더 설득력 있음
- 정확한 예언보다 적용 가능한 해석에 대한 수요 증가

### 2.4 경쟁 환경 분석
| 경쟁 요소 | 기존 서비스 특징 | NingNing 차별화 |
|-----------|-----------------|----------------|
| 정통성 연출 | 만세력 기반이나 결과가 단순 | 사주 구조를 시각적으로 보여주고 AI가 해설 |
| 가벼운 입구 | 테마별 운세 제공 | 검색형 랜딩 + 테마 진입 적극 활용 |
| 반복 방문 | 매일 운세, 행운 키워드 | 오늘의 총평 + AI 질문 + 저장/재열람 |
| 상담 확장 | 전문가 연결 | AI 질문 흐름으로 즉시 확장 |
| 결제 구조 | 구독 중심 | 단건 심층 리포트 우선 |

## 3. 비즈니스 모델

### 3.1 수익 구조

#### 무료 (Free)
- 사주 입력
- 오늘의 총평
- 연애/일/재물 3축 요약
- AI 추가질문 1~3회

#### 심층 리포트 (3,900 ~ 7,900원)
- 오늘의 흐름 상세 해석
- 해석 배경 설명 (왜 이런 흐름인지)
- 구체적 행동 조언
- 저장 및 재열람

#### AI 질문 팩 (1,900 ~ 4,900원)
- 추가 질문 횟수 확장
- 테마별 질문 팩 (연애, 이직, 재물 등)

### 3.2 가격 전략
- 초기에는 단건 상품 1개로 시작
- 가격보다 '왜 사야 하는지' 메시지가 먼저
- 무료 영역과 유료 영역 경계를 명확히 구분
- 구독 모델은 PMF 확인 후 검토

### 3.3 수익 시나리오 (출시 후 6개월 기준)

| 지표 | 보수적 | 기본 | 낙관적 |
|------|--------|------|--------|
| 월 방문자 (UV) | 30,000 | 80,000 | 200,000 |
| 결과 화면 도달율 | 60% | 75% | 85% |
| 결제 전환율 | 1.5% | 2.5% | 4.0% |
| 평균 객단가 | 4,900원 | 5,500원 | 6,500원 |
| 월 매출 | 약 132만원 | 약 825만원 | 약 4,420만원 |

## 4. 마케팅 전략

### 4.1 채널 전략
- **1차**: 모바일 웹 (검색 유입 최적화)
- **2차**: SNS 공유 (결과 이미지 카드)
- **3차**: 시즌별 캠페인 (신년운세, 이직 시즌, 발렌타인 등)

### 4.2 유입 전략
| 단계 | 전략 | 실행 |
|------|------|------|
| 인지 | 테마별 검색 랜딩 | "2026 이직운", "오늘 연애운" 등 SEO |
| 관심 | 샘플 결과 노출 | 결과 미리보기 + 입력 유도 |
| 전환 | 결과 화면 내 CTA | 자연스러운 질문→결제 흐름 |
| 재방문 | 오늘의 총평 + 알림 | 매일 다른 해석, 저장 기능 |

### 4.3 콘텐츠 마케팅
- 테마별 랜딩 페이지 2~3종 (연애운, 이직운, 재물운)
- 결과 공유용 이미지 카드 생성
- 시즌성 콘텐츠 (신년운세, 궁합 시즌)

## 6. 리스크 관리

### 6.1 주요 리스크
| 리스크 | 영향도 | 대응 방안 |
|--------|--------|----------|
| 결과 품질 저하 | 높음 | 프롬프트 버전 관리, 정기 검수, 가드레일 |
| AI 응답 속도 지연 | 높음 | 결과 생성 5초 이내 목표, 캐싱 전략 |
| 과장/공포 메시지 | 중간 | 금지 표현 필터, 톤 가이드라인 |
| 개인정보 이슈 | 높음 | 최소 수집 원칙, 삭제 동선, 처리방침 |
| 기능 과다 (스코프 크립) | 중간 | MVP 범위 엄수, 후순위 명확 구분 |

### 6.2 법적·정책 체크
- 개인정보 처리방침 준비
- 환불/문의 동선 설계
- 계정 삭제 안내
- 결제 실패 대응 프로세스
- 의료·법률·투자·자해 관련 조언 금지

## 7. 성장 로드맵

### Phase 1: MVP 검증 (1~3개월)
- 사주 기반 해석 경험 하나에 집중
- 웹 MVP 출시, 첫 결과 화면 완성도 확보
- 결제가 되는 리포트 구조 발견

### Phase 2: 초기 성장 (4~6개월)
- 테마 랜딩 확장 (연애, 이직, 재물)
- 공유 이미지 카드, 알림 옵트인
- 카피/가격 실험 반복

### Phase 3: 확장 (7~12개월)
- 궁합 서비스 추가
- 신년운세/시즌 패키지
- 앱 패키징 검토

### Phase 4: 플랫폼화 (1년~)
- 타로·관상 등 카테고리 확장
- 전문가 상담 연결
- 구독 모델 도입 검토

---

## 8. 핵심 의사결정

1. **코어**: 첫 버전은 사주 하나로 간다
2. **채널**: 모바일 웹부터 검증한다
3. **화면**: 가장 강한 화면은 홈이 아니라 결과 화면이다
4. **수익**: 구독보다 단건 상품부터 시작한다
5. **신뢰**: 정통성은 계산에서, 매력은 해석 문장에서 만든다

---

*이 MVP의 성공 기준은 '운세 기능이 많다'가 아니라, 첫 결과를 본 사용자가 더 묻고 싶어지고, 결국 결제까지 납득되는가이다.*
